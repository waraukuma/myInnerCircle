# 1인 프로젝트 홈페이지 문서 패키지

본 문서는 **1인 개발 프로젝트 홈페이지**를 개발하면서 함께 제출하거나 포트폴리오로 활용할 수 있도록 구성된 표준 문서 모음입니다.
(기획 → 설계 → 구현 → 운영 흐름을 모두 설명 가능)

## 1. 프로젝트 개요서

### 1.1 프로젝트명

## 1. 프로젝트 개요서

### 1.1 프로젝트명

내사람 관리 – 지인 인사 관리 및 기호 파악 서비스

### 1.2 프로젝트 목적
* 지인, 가족, 거래처 등의 **인사 관리(생일·기념일)**를 체계적으로 관리
* 선물 선택 시 상대방의 **기호·비선호 정보를 기록**하여 실수 방지
* 상황에 맞는 선물/연락 아이디어를 제공하는 개인 맞춤형 관리 서비스 구현

### 1.3 개발 배경
* 명절, 생일, 경조사 등에서 무엇을 해야 할지 잊거나 고민하는 상황 빈번
* 메모 수준을 넘어 **체계적인 인사·관계 관리 서비스** 필요성 인식
* CRUD, 권한, 상태값, 추천 로직 등 실무형 기능을 포함한 1인 프로젝트 목적

### 1.4 대상 사용자
* 인사 관리가 필요한 일반 사용자
* 영업·관리 업무를 수행하는 직장인
* 관계 관리를 중요하게 생각하는 개인 사용자

## 2. 요구사항 정의서
### 2.1 기능 요구사항

#### 공통
* 회원가입 / 로그인 / 로그아웃
* 비밀번호 암호화 처리
* 로그인 사용자만 데이터 접근 가능

#### 사용자 기능
* 지인 등록 / 수정 / 삭제
* 관계 유형 관리 (가족, 친구, 직장, 거래처 등)
* 기념일 등록 (생일, 결혼기념일, 명절 등)
* 기호 정보 관리 (좋아함 / 싫어함 / 주의사항)
* 인사 이력 관리 (언제, 무엇을 했는지 기록)

#### 알림 기능
* 기념일 임박 시 알림 표시
* 최근 인사 미진행 대상 표시

#### 관리자 기능
* 사용자 관리
* 서비스 데이터 통계 조회

### 2.2 비기능 요구사항
* 개인정보 보호를 위한 사용자별 데이터 분리
* 예외 상황 공통 처리
* 확장 가능한 테이블 설계

## 4. DB 설계서
### 4.1 테이블 목록
* USER
* CONTACT
* ANNIVERSARY
* PREFERENCE
* GREETING_HISTORY

### 4.2 CONTACT 테이블
| 컬럼명           | 타입      | 설명     |
| ------------- | ------- | ------ |
| contact_id    | bigint  | PK     |
| user_id       | bigint  | 소유 사용자 |
| name          | varchar | 지인 이름  |
| relation_type | varchar | 관계 유형  |
| memo          | text    | 비고     |

### 4.3 PREFERENCE 테이블
| 컬럼명        | 타입      | 설명             |
| ---------- | ------- | -------------- |
| pref_id    | bigint  | PK             |
| contact_id | bigint  | FK             |
| category   | varchar | 음식/선물/취향       |
| content    | varchar | 내용             |
| like_type  | varchar | LIKE / DISLIKE |

## 5. API 설계서

| Method | URL                | 설명       |
| ------ | ------------------ | -------- |
